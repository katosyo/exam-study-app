# Git Rules（Release）

## 基本方針

本プロジェクトは **正式リリースフェーズ** として、
変更内容・理由・影響範囲を常に追跡可能にする。

- main ブランチは常にデプロイ可能であること
- 直接の変更は禁止する

---

## ブランチ運用

- 作業は必ず main からブランチを切る
- ブランチは **1機能 / 1修正単位** とする

命名例：
- feature/quiz-selection
- fix/result-calculation
- chore/update-deps

---

## プルリクエスト（PR）ルール

- すべての変更は PR 経由で main に取り込む
- PR は **必ず自分でレビューしてからマージ** する

PR には以下を必須とする：

- 何を変更したか
- なぜ変更したか
- 影響範囲（UI / API / Data など）
- リリース可否（Yes / No）

---

## コミットルール

- 1コミット1意味
- 「何をしたか」ではなく「なぜしたか」を書く
- 自動生成コミットは禁止

---

## マージルール

- squash merge を原則とする
- main の履歴は PR 単位で意味が分かる状態を保つ

---

## 禁止事項

- main への直接 push
- レビューなしマージ
- 変更内容が説明できない PR
